# Architecture

This page documents the architecture of the currently shipped plugin runtime.

## Project Structure

```text
iina-plugin-bookmarks/
├── src/
│   ├── index.ts                    # Plugin entry point
│   ├── bookmark-manager.ts         # Core orchestrator
│   ├── bookmark-persistence.ts     # Preferences-backed storage
│   ├── bookmark-import-export.ts   # JSON/CSV import + export
│   ├── thumbnail-generator.ts      # Thumbnail generation
│   ├── types.ts                    # Runtime/type contracts
│   └── utils/
│       ├── formatTime.ts
│       └── validation.ts
├── ui/
│   ├── sidebar/
│   ├── overlay/
│   ├── window/
│   └── components/
├── tests/
└── docs/
```

## Runtime Design

`BookmarkManager` is the central orchestrator. It wires IINA runtime dependencies to three UI
surfaces (sidebar, overlay, window), then delegates specialized work to focused modules:

- `BookmarkPersistence` for load/save/recovery
- `BookmarkImportExport` for import/export and duplicate policies
- `ThumbnailGenerator` for optional thumbnail assets

## UI Communication Model

UI surfaces communicate with the backend through message passing (`postMessage` / `onMessage`).
`BookmarkManager` handles messages like:

- bookmark CRUD (`ADD_BOOKMARK`, `UPDATE_BOOKMARK`, `DELETE_BOOKMARK`)
- navigation (`JUMP_TO_BOOKMARK`, `NEXT_BOOKMARK`, `PREV_BOOKMARK`)
- import/export (`IMPORT_BOOKMARKS`, `EXPORT_BOOKMARKS`)
- reconciliation (`RECONCILE_FILES`, `FILE_RECONCILIATION_REQUEST`)
- utility events (`UI_READY`, `REQUEST_FILE_PATH`, `REQUEST_BOOKMARK_DEFAULTS`)

## Data Model Notes

Bookmarks are persisted in plugin preferences and include annotation fields such as:

- `tags`, `description`
- optional `color`, `pinned`, `endTimestamp`
- optional enrichment fields (`chapterTitle`, `subtitleText`, `thumbnailPath`, `scratchpad`)

Collections and smart collections are also persisted by `BookmarkManager`.

## Reliability and Safety

Current runtime protections include:

- input sanitization (`stripHtmlTags`) on user-controlled text
- timestamp and end-range validation
- CSV formula-injection mitigation on export
- duplicate detection on add/import
- backup-and-recover path in persistence

## Shipped vs Roadmap

</Callout>
